# Contributing

Thanks for contributing!

## Development setup
- Use a Home Assistant dev container or your preferred local environment.
- Keep changes focused and documented.

## Pull requests
- Keep PRs small and scoped.
- Include screenshots/logs if UI or behavior changes.
- Ensure GitHub Actions pass (hassfest + hacs).

## Coding style
- Prefer readability over cleverness.
- Avoid vendor-specific assumptions unless guarded by detection/fallback.
